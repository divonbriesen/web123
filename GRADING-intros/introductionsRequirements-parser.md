# Introductions Requirements Parser Version

This file is a stricter, machine-friendly version of the same checklist.

## 1. Placement and Heading

- Rule: Entry is alphabetized by last name.
- Rule: Entry starts with a blank line below a horizontal rule.
- Check: Entry title is Heading 2.
- Check: Only the student name line should use heading formatting; non-name content should not be a heading.
- Check: A horizontal rule appears above the entry heading.
- Check: At least one blank paragraph appears between that horizontal rule and the entry heading.
- Check: Heading 2 text matches `Last Name, First Name` with an optional trailing middle initial.
- Check: Heading 2 name line is left-aligned (not centered).
- Check: Entries are in alphabetical order by last name.

## 2. Public Acknowledgment

- Rule: A public acknowledgment sentence is required.
- Check: A sentence about the post being public is present.
- Check: The acknowledgment includes initials and a date.
- Check: The acknowledgment is left-aligned normal text.

## 3. Display Name Line

- Rule: A display name line is required.
- Check: The line matches `First M. "Nickname" Last ~ Adjective Animal`.
- Check: The middle initial is optional.
- Check: The nickname is optional.
- Check: The separator before the animal phrase is one of `~`, `|`, or `*`.
- Check: The line is large and centered, but not a heading.

## 4. Photo and Caption

- Rule: A centered photo is required.
- Check: A photo is present.
- Check: The photo is centered.
- Rule: A caption is required under the photo.
- Check: The caption is italic.
- Check: The caption is directly under the photo.
- Check: The caption is concise.

## 5. Personal Statement

- Rule: A personal statement is required.
- Check: The statement has 3 or more complete sentences.
- Check: The statement is normal left-aligned body text.

## 6. Required Background Bullets

- Rule: Background content must use bullet points.
- Check: Bullet points are present.
- Rule: The text before each colon must be bold.
- Check: Required labels are bold before the colon.
- Required labels:
  - Personal Background:
  - Professional Background:
  - Academic Background:
  - Primary Work Computer:
  - Operating System & Version:
  - Backup Work Computer & Location Plan:
  - Courses I'm Taking, & Why:

## 7. Courses Section Structure

- Rule: The Courses I'm Taking, & Why: item must contain nested bullets.
- Check: Nested course bullets are present under that item.
- Rule: Each nested bullet must include a course code, course name, colon, and reason.
- Check: Format example: `ITIS3135 - Front-End Web Development: reason for taking the course.`

## 8. Optional Bullets

- Optional label: Funny/Interesting Item to Remember Me by:
- Optional label: I'd Also Like to Share:
- Check: Remove the bullet entirely if the optional item is not included.

## 9. Favorite Quote

- Rule: A favorite quote is required.
- Check: The quote appears after exactly one blank line.
- Check: The quote is on one line.
- Check: The attribution is on the next line.
- Check: The attribution starts with `-` or `~`.

## 10. Spacing and Formatting Rules

- Rule: Leave exactly one blank line before the quote.
- Rule: No instances of multiple consecutive spaces are allowed in visible text content.
- Check: No extra blank lines appear elsewhere.
- Check: No content is centered with spaces.
- Check: No multiple consecutive spaces appear between visible characters.
- Check: No underlining appears except real links.

## 11. Required Links

- Rule: These links are required only for web classes (ITIS3135, WEB115, WEB215, WEB250). CIS110 students should not be graded on this links requirement.
- Rule: Leave exactly one additional blank line after the quote attribution before the links line.
- Rule: A centered links line is required below the quote block.
- Check: The links line includes all of the following labels:
  - CLT Web
  - GitHub
  - GitHub.io
  - freeCodeCamp
  - Codecademy
  - LinkedIn
- Check: The links appear on one centered line with a divider between each link (example divider: `|`); with 6 required links, this is at least 5 dividers.
- Check: `CLT Web` links to `https://webpages.charlotte.edu/xy...` where `x` and `y` are first/last initials and remaining characters are alphanumerics.
- Check: `GitHub.io` links to `https://xyz.github.io`
- Check: `GitHub` links to `https://github.com/xyz` where `xyz` is the same as those above.
- Check: `freeCodeCamp` links to the student's freeCodeCamp profile page (for example: `https://www.freecodecamp.org/<profile-id>`).
- Check: `Codecademy` links to the student's Codecademy profile page (for example: `https://www.codecademy.com/profiles/<profile-id>`).
- Check: `LinkedIn` links to the student's LinkedIn profile page.

## 12. Entry Separator

- Rule: After each student's entry, include at least one blank line and a horizontal rule.
- Check: At least one blank paragraph appears before the separator.
- Check: A horizontal rule (`<hr>`) appears before the next Heading 2 entry.
