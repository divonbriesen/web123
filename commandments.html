Web Coding Commandments

Web Standards

Use these commandments as the baseline standard for every assignment, demo, and revision. The goal is simple: write code that is readable, valid, maintainable, and easy for another developer to understand.

Required Commandments

  1. Files shall not have any spaces or capital letters.
  2. Every page shall have header, main, and footer elements, and use section, article, and nav where they fit the content, with nav in the header if it is not a standalone element.
  3. Pages that are part of the same site shall have matching headers and matching footers.
  4. There shall be one h1, reflecting the site name at the start of the header element, one h2, reflecting the page name as the first element in the main element, and sections within main shall start with h3 elements. The title element shall comprise of the h1 + divider + h2.
  5. Styles will be in a styles folder, and the default stylesheet will be called default.css.
  6. Images will be in your images folder and have clear names indicating what they are.
  7. Each page shall have an obvious validator.
  8. All validations should pass without errors or warnings, even those requiring manual checking.
  9. All pages should follow WCAG guidelines.
  10. Avoid spans, divs, and classes where normal tags or elements will do.
  11. Learn and follow the CRAP design principles: Contrast, Repetition, Alignment, and Proximity.
  12. Test test test- test readabililty, test on different devices, test with your grandma.
